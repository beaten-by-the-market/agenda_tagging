HWPX 안건 태깅 시스템 — 사용법

1. 한눈에 보기 — 작업 흐름

① 열기상단 HWPX 열기로 파일 로드
→
② 항목 선택좌측 카드의 선택 버튼
→
③ 영역 잡기원문에서 클릭 또는 드래그
→
④ 저장HWPX 저장 또는 JSON 내보내기

좌측 항목 하나마다 본문에서 근거가 되는 영역을 여러 개 잡아서 묶을 수 있고, 결과는 한컴 누름틀(fieldBegin/fieldEnd) 또는 JSON으로 저장됩니다.

2. 좌측 패널 — 태그대상 항목

  • 현재 표준: 카드 위쪽에 YYYY-MM-DD 기준으로 표시됩니다. 표준 자체는 수정으로 변경하고, 변경 이력은 이력에서 확인.
  • 각 항목 카드는 다음 상태를 가집니다:
    • 아직 잡힌 영역이 없음 → 선택
    • 잡힌 영역이 있음 → 초기화 추가
    • 활성 중(영역 잡는 중) → 선택 중 / 추가 중
  • 미리보기: 잡힌 영역들이 1. [문단] ... / 2. [셀] ... 같은 번호 리스트로 표시. 6개를 넘으면 자동 접힘.

3. 우측 원문 — 영역 잡는 방법

3-1. 클릭 (단일 영역)

  • 일반 문단 위 클릭 → 그 문단 통째.
  • 셀 안 문단 위 클릭 → 그 문단. (단, 셀에 문단이 하나뿐이면 자동으로 셀 전체로 승격됨.)
  • 셀 여백/패딩 클릭 → 그 셀 전체.

3-2. 드래그 (여러 영역 한 번에)

드래그가 어디에 걸치느냐에 따라 다음 4가지 규칙으로 자동 판정됩니다.

CASE 1 · 한 문단 안
→ 그 문단 1개
○ …드래그… 끝남 다음 문단은 그대로
CASE 2 · 여러 문단 가로지름
→ 각 문단 따로 (N개)
○ 첫 문단 전체 ○ 둘째 문단 전체 ○ 셋째 문단 전체
CASE 3 · 한 셀 안 여러 문단
→ 그 셀 전체 (1개)
┌─────────────┐ │결의일 │ │납입일 │ │조달금액 │ └─────────────┘
CASE 4 · 여러 셀 또는 표 안↔밖
→ 표 전체 (모든 셀 N개)
┌─────┬─────┐ │A1 │B1 │ ├─────┼─────┤ │A2 │B2 │ └─────┴─────┘
요점: 셀과 문단이 섞이면 항상 "더 큰 단위로 승격"됩니다. 한 셀 안 단일 문단을 정밀히 잡고 싶을 때는 그 셀에 문단이 2개 이상이어야 합니다.

4. 모드 — 선택 · 추가 · 초기화

버튼동작언제 쓰나
선택새로 영역을 잡음. 기존이 있었다면 교체.처음 잡을 때 / 다시 잡고 싶을 때 (초기화 후)
추가기존 영역은 유지하고 새 영역을 덧붙임.다른 페이지/문단을 같은 항목에 더 묶고 싶을 때
초기화그 항목의 모든 영역을 비움.잘못 잡았을 때 / 처음부터 다시 할 때
중복 자동 정리: "추가"로 셀과 그 안의 문단이 동시에 잡히면, 문단은 자동 제거되어 셀만 남습니다.

5. 표준 항목 관리

좌측 표준 바의 수정을 누르면 항목 자체를 변경할 수 있습니다.

  • 항목 추가: + 항목 추가로 새 행, 영문 ID(필수)와 표시 이름 입력.
  • 항목 삭제: 행의 삭제 토글. 잘못 눌렀으면 같은 자리에서 복원.
  • 이름 변경: 표시 이름을 그 자리에서 수정.
  • 표준 일자 · 수정자 · 메모: 누가 언제 왜 바꿨는지 같이 기록.
  • 저장하면 자동으로 이력에 등재 + localStorage에 보관.

이력 버튼으로 시간순 변경 내역(추가/삭제/수정)을 색상 구분된 박스로 확인할 수 있습니다.

6. 표준 공유 — JSON 가져오기/내보내기

표준 자체를 다른 PC/동료와 맞추려면 수정 모달 하단의:

  • JSON 내보내기 — 현재 표준(항목 + 전체 이력)을 agenda-tagging-standard_YYYY-MM-DD.json 파일로 저장.
  • JSON 불러오기 — 받은 JSON 파일을 적용 (현재 localStorage 표준 덮어쓰기).

7. 결과 파일 — 무엇이 어떻게 저장되나

HWPX (누름틀)

각 잡은 영역(unit)이 하나의 누름틀이 됩니다. 같은 항목이 여러 영역이면 같은 prefix + 번호.

<hp:fieldBegin id="..." type="CLICK_HERE" name="companyName_1" ...>
  <hp:parameters>
    <hp:stringParam name="Direction">기업명 (1/3)</hp:stringParam>
  </hp:parameters>
  <hp:metaTag>{
    "schema":"agenda-tagging/v1",
    "tagId":"companyName", "label":"기업명",
    "partIndex":1, "partCount":3,
    "unitType":"tc"
  }</hp:metaTag>
</hp:fieldBegin>
   …태깅된 본문…
<hp:fieldEnd beginIDRef="..." />

JSON

{
  "fileName": "샘플서식.hwpx",
  "taggedAt": "2026-05-11T...Z",
  "tags": {
    "기업명": [
      { "partIndex":1, "partCount":3, "type":"p",  "text":"…" },
      { "partIndex":2, "partCount":3, "type":"tc", "text":"…" },
      { "partIndex":3, "partCount":3, "type":"tc", "text":"…" }
    ],
    "사유발생일": [ ... ],
    ...
  }
}

필드 의미

필드의미
tagId표준 항목의 영문 ID
label표준 항목의 표시 이름
partIndex / partCount같은 항목 내 몇 번째 / 총 몇 개 영역인지
type / unitTypep=문단, tc=셀

8. 자주 발생하는 상황

  • 표를 가로지른 영역이 너무 크게 잡힘 → 의도적입니다(표 전체로 확장). 셀 하나만 잡고 싶으면 그 셀 안에서만 드래그·클릭.
  • 잡으니 [문단]이 아니라 [셀]로 잡힘 → 그 셀에 문단이 하나뿐이라 자동 승격된 것. 동치이므로 보통 그대로 사용해도 무방.
  • 같은 항목을 여러 곳에 흩어 잡고 싶다 → 추가로 누적. 페이지/섹션이 달라도 OK.
  • 저장 메시지의 필드 개수와 실제가 다르다 → 콘솔에 "삽입 실패" 경고가 있는지 확인. 보통 셀 안 문단 매칭 실패가 원인입니다.
